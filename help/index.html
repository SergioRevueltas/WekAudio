Overwiew

This application is a prototype tool for audio pattern recognition by artificial intelligence. The main function can be summarized in two stages: training and classification.

Training

The training is the process that consists of loading, pre-processing, analysis and feature extraction of audio files indicating that class or type are. From that process it is obtained a model based on a classifier (Bayesian network, decision tree, ...) that subsequently allow recognize new patterns of these audio classes.

In another words, it comes to teaching the application to recognize particular audio signals or patterns like bells, horns, hooters, animals, instruments, human voices… by loading a lot of audio patterns of those classes to gain the knowledge that will allow classify and identify them in the future.

To train the application properly, we have to load at least 2 classes of audio patterns (e.g., siren and bell) to discern between them, of which we must load sufficient number of audio files (10 total at least). It is obvious to say that while we load more files of each class, training and classification will be more accurate.

In this application, training process consists of 3 steps:

  1. RECORDINGS panel: To load audio files
  2. FEATURES panel: To choose features, compression method (Aggregator) and Analysis options.
  3. DATA MINING panel: To choose classifier and to train.

1. Load audio files

The audio file loading for training process has to accomplish some restrictions:

  • You must add at least 10 audio files.
  • You must add at least 2 classes of sounds or audio patterns.
  • The audio file format must be wave (.wav) with the following nomenclature:

    className + (blank) + anything.wav


    For example, some valid file names:


    “dog 00.wav”, “dog 001.wav”, “dog 00002.wav”, “dog qwerty.wav”…


    class1 = dog


    “cat (0).wav”, “cat (01).wav”, “cat (245).wav”, “cat (3654).wav”…


    class2 = cat

NOTE: You can rename a group of files at once. In Windows, for example, just select all files that you want rename, by clicking on them while you hold CTRL or SHIFT key, and press F2 key to rename them. The OS will assign the same name to all selected files with an auto increment integer concatenated at the end.

There is no restrictions in terms of quality, sample rate, bit depth or audio file length, even though it is recommended short length audio files free of noise.

It should be noted that while we load more files of each class, the model that will we get from training will be better.

  • You can add audio files by pressing “Add Recordings” button or from the File menu, which allow multiple file selection. The selected files will be added to RECORDINGS panel table.
  • You can record from microphone and add it by pressing “Add from mic” button. It will open a small recorder that allows record audio from microphone with different encoding formats and save it to use for training. The selected file will be added to RECORDINGS panel table.
  • You can play loaded audio files by double clicking on them in RECORDINGS panel table.
  • You can delete loaded audio files by selecting them and pressing “Delete Recordings” button. It allows multiple selections.

2. Feature selection

The feature selection is the most complex part of the application, inasmuch as the correct choice of them will depend on what patterns we want to identify, among many other factors that would require extra education in this area (type of sound-wave, tone, pitch, intensity, timbre…)

Therefore, there have been preselected a set of features to be extracted in FEATURES panel table, with which is obtained highly representative of the original audio signal, as well as a method of data compression (Aggregator) and analysis options by default.

However, although there is the possibility to select the features that you want even edit settings of some of them, it is recommended not select too many if not are known or studied in depth with the Weka tool.

  • You can select the features by clicking on “save” column checkboxes from FEATURES panel table.
  • You can obtain information about features even edit some of them by double clicking.
  • You can choose the analysis compression method (Aggregator) by pressing on “Alter Aggregators” button from FEATURES panel.
  • You can edit analysis options by pressing on “Analysis Options” button form FEATURES panel which allows set sample rate, window size, window overlap and normalize data or not.

3. Train

In this step we just have to choose a classifier and training set save path (.arff) in DATA MINING panel, which will contains the selected features of preloaded audio files.

When you press “Train” button in DATA MINING panel, the application analyzes all loaded audio files (with selected analysis options, extraction features and aggregators) and generates 2 files with same name but different extensions:

  • Weka file (.arff)
  • Classifier file (.model)

This process can take several minutes. It will vary depending on the experiment to be performed.

After build the model, WekAudio does a cross validation evaluation and print the summary results in main window, including accuracy, hit rate or confusion matrix among others.

Classification

The classification is the process that consists of loading, pre-processing, analysis and feature extraction of audio files without indicating what class or type are, so the application is able to determine each class using a previously stored model or classifier.

In this application, classification process consists of 3 steps:

  1. FEATURES panel: To choose features, compression method (Aggregator) and Analysis options.
  2. DATA MINING panel: To load an unknown audio file and previously stored model or classifier.
  3. DATA MINING panel: Classify.

1. Feature selection

The feature selection has to be the same that you did in training process. In another words, it has to be the same features that were stored in saved model or classifier.

2. Load audio and model files

The audio file format for classification process must be wave (.wav).

There are no restrictions about file nomenclature.

There are no restrictions in terms of quality, sample rate, bit depth or audio file duration, even though it is recommended short duration audio files and free of noise.

Additionally, in this step you have to select the model path, previously generated in the training process, which will contain the selected features data of trained audio files.

3. Classify

When you press the classify button in DATA MINING panel, the application analyzes loaded audio file with selected analysis options, features and aggregators settings, and classify it with the model that was selected in previous step. Finally, WekAudio will print classification result on main window.

This process can take several minutes. It will vary depending on the experiment to be performed.
